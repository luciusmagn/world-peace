#+title: World Peace

World Peace is a tiny no-types language implemented on SBCL. The repository
contains the interpreter, the image-dumping compiler, examples, and editor
support.

The language spec lives in [[file:world-peace.org][world-peace.org]].

* Requirements

- SBCL
- POSIX shell for =check-lisp=
- =curl= only if a program uses =load some.host/path/file.wp;=

Quicklisp is not required. The system has no external Lisp dependencies; it
only uses ASDF/UIOP, which are bundled with SBCL.

* Build

From the repository root:

#+BEGIN_SRC shell
sbcl --non-interactive \
  --eval '(require :asdf)' \
  --eval '(asdf:load-asd (merge-pathnames "world-peace.asd" (uiop:getcwd)))' \
  --eval '(asdf:make :world-peace)'
#+END_SRC

This writes the =peace= executable into the repository root.

* Nix

Build the package from a checkout:

#+BEGIN_SRC shell
nix build .#world-peace
./result/bin/peace run examples/tour/01-hello.wp
#+END_SRC

Run it without installing:

#+BEGIN_SRC shell
nix run .# -- run examples/tour/01-hello.wp
nix run .# -- repl
#+END_SRC

Install it into your user profile from a checkout:

#+BEGIN_SRC shell
nix profile install .#world-peace
peace repl
#+END_SRC

Install it directly from GitHub:

#+BEGIN_SRC shell
nix profile install github:luciusmagn/world-peace#world-peace
peace repl
#+END_SRC

The installed package includes the =peace= executable, the spec under
=share/doc/world-peace/=, examples under =share/world-peace/examples/=, and
editor support under =share/world-peace/editors/=. For example:

#+BEGIN_SRC shell
peace run ~/.nix-profile/share/world-peace/examples/tour/01-hello.wp
#+END_SRC

For development, enter a shell with SBCL and curl available:

#+BEGIN_SRC shell
nix develop
./check-lisp
#+END_SRC

The flake also exposes the Neovim runtime as a plugin package:

#+BEGIN_SRC nix
programs.neovim.plugins = [
  inputs.world-peace.packages.${pkgs.system}.world-peace-neovim
];
#+END_SRC

If your Nix installation has flakes disabled, add:

#+BEGIN_SRC shell
--extra-experimental-features 'nix-command flakes'
#+END_SRC

to the =nix= commands above.

* Run Programs

Run a source file directly:

#+BEGIN_SRC shell
./peace run examples/tour/01-hello.wp
#+END_SRC

Run a source root plus entrypoint explicitly:

#+BEGIN_SRC shell
./peace run examples/tour/08-loads main.wp
#+END_SRC

Start the REPL:

#+BEGIN_SRC shell
./peace repl
#+END_SRC

* Compile Programs

Dump a standalone SBCL image for a program:

#+BEGIN_SRC shell
./peace compile examples/tour/01-hello.wp -o /tmp/world-peace-hello
/tmp/world-peace-hello
#+END_SRC

The two-argument source-root form also works:

#+BEGIN_SRC shell
./peace compile examples/tour/08-loads main.wp -o /tmp/world-peace-loads
/tmp/world-peace-loads
#+END_SRC

* Syscall Buffers

=syscall= accepts numbers, arrays, and byte buffers. Numbers are passed as
integer syscall arguments. Arrays are copied into temporary byte buffers for the
duration of the call. =bytes(n)= creates a mutable byte buffer that is passed as
its native pointer, and kernel writes are visible afterward.

Useful builtins:

- =bytes(n)= creates a zero-filled mutable byte buffer.
- =poke(buffer, index, value)= writes one byte.
- =addr(buffer)= returns the native pointer as a number, for explicit pointer
  nonsense. Keep the buffer value alive while using this number.

Linux/x86-64 example:

#+BEGIN_SRC shell
./peace run examples/linux/syscall-buffer.wp
#+END_SRC

* Test

Run the Lisp test suite:

#+BEGIN_SRC shell
./check-lisp
#+END_SRC

The tests load the local ASDF system directly and do not use Quicklisp.

* Editor Support

Emacs support lives in =editors/emacs=. Add that directory to =load-path= and
=require= =world-peace-mode=.

Neovim support lives in =editors/neovim=. From a checkout, add it to your
runtime path:

#+BEGIN_SRC lua
vim.opt.runtimepath:prepend("/path/to/world-peace/editors/neovim")
#+END_SRC

Or, with Vimscript:

#+BEGIN_SRC vim
set runtimepath^=/path/to/world-peace/editors/neovim
#+END_SRC

With a Nix profile install, the runtime is available here:

#+BEGIN_SRC lua
vim.opt.runtimepath:prepend(vim.fn.expand("~/.nix-profile/share/world-peace/editors/neovim"))
#+END_SRC

The Neovim runtime provides =*.wp= filetype detection, syntax highlighting,
comment settings, indentation, and insert-mode =<CR>= continuation for =--- =
body lines.
